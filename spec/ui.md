# UI設計書

## 共通仕様

### レイアウト
- 左サイドバー（幅240px、折りたたみ可能）+ メインコンテンツ領域
- サイドバーにはロゴ、ナビゲーションメニュー、ユーザー情報を配置
- Notionスタイルのミニマムで洗練されたデザイン
- カラーパレット: 白背景(#FFFFFF)、テキスト(#37352F)、アクセント(#EB5757 不正検知用赤)、セカンダリ(#2F80ED 青)、ボーダー(#E9E9E7)
- フォント: Inter（英数字）、Noto Sans JP（日本語）

### サイドバーナビゲーション
- ネットワーク可視化（アイコン: グラフノード）
- アラート一覧（アイコン: ベル + 未対応件数バッジ）
- パターン分析（アイコン: チャート）
- 区切り線
- 設定（アイコン: 歯車）

### ローディング
- 全ページ共通でSVGアニメーションによるローディング表示
- ページ遷移時: 中央に配置したパルスアニメーション（円が波紋のように広がる）
- データ取得中: コンテンツ領域にスケルトンスクリーン + 上部にプログレスバー

### レスポンシブ対応
- 最小幅1280px（デスクトップ業務端末を想定）
- サイドバーは折りたたみ可能（アイコンのみ表示に切り替え）

## ページ1: 取引ネットワーク可視化ページ

パス: `/network`

### 概要
口座間の資金フローをインタラクティブなネットワークグラフで表示し、不正取引の経路を視覚的に把握するページ。

### ヘッダーエリア
- ページタイトル「取引ネットワーク」
- 口座検索バー: テキスト入力で口座IDを検索。オートコンプリート付き。Enterまたは検索ボタンで指定口座を起点にグラフを描画
- ホップ数セレクター: ドロップダウンで1〜5ホップを選択（デフォルト: 2）
- 期間フィルター: 日付範囲ピッカー（開始日・終了日）

### ネットワークグラフエリア（メインコンテンツの80%幅）
- 力学モデルベースのインタラクティブグラフ
- ノード: 口座を表す円。サイズは取引金額の合計に比例
  - 色: 不正スコア高（赤 #EB5757）、中（黄 #F2C94C）、低（グレー #BDBDBD）
  - ホバー時: 口座ID、銀行名、エンティティ名をツールチップ表示
  - クリック時: 口座プロファイルページ（1.2）へ遷移
- エッジ: 取引を表す線。太さは取引金額に比例
  - 色: 不正フラグ付き取引は赤、正常取引はグレー
  - 矢印で送金方向を表示
  - ホバー時: 金額、通貨、日時、支払手段をツールチップ表示
  - クリック時: 取引詳細ページ（1.1）へ遷移
- ズーム・パン操作対応（マウスホイール、ドラッグ）
- 凡例: 右下に配置。ノード色・エッジ色の意味を表示

### パターン検出パネル（メインコンテンツの右サイド、幅280px）
- 検出されたパターン一覧をカード形式で表示
- 各カード: パターン種別（FAN-OUT, CYCLE等）、関連取引数、合計金額
- カードクリック時: 該当パターンの取引をグラフ上でハイライト

## ページ1.1: 取引詳細ページ

パス: `/transactions/:transactionId`

### 概要
個別取引の詳細情報と、モデルの判定根拠を表示するページ。

### 取引情報カード
- 2カラムのグリッドレイアウトで以下を表示:
  - 左カラム「送金元」: 銀行名（Bank ID → accounts.csvで名前解決）、口座ID、エンティティ名
  - 右カラム「送金先」: 同上
  - 矢印アイコンで送金方向を示す
- 下部に取引メタ情報を横並びで表示:
  - 取引日時
  - 支払金額 / 支払通貨
  - 受取金額 / 受取通貨
  - 支払手段
- 不正フラグ: Is Laundering = 1 の場合、赤いバッジ「不正検知」を目立つ位置に表示

### モデル判定セクション
- 不正確率スコア: 大きな数値表示（例: 「87.3%」）+ 円形プログレスバーで視覚化
- 判定結果: 「不正の疑いあり」/ 「正常」をラベル表示
- 特徴量重要度: 横棒グラフで上位5つの寄与特徴量を表示（例: Amount Paid, Payment Format_ACH, ...）

### 同一口座の取引履歴セクション
- タブ切り替え:「送金元口座の履歴」/「送金先口座の履歴」
- 各タブ内にテーブル表示:
  - カラム: 日時、相手銀行、相手口座、金額、通貨、支払手段、不正フラグ
  - 不正フラグ付き行は背景を薄赤(#FFF5F5)でハイライト
  - 行クリックで該当取引の詳細ページへ遷移
- ページネーション: 1ページ20件

### 担当者メモセクション
- テキストエリア（最大500文字）
- 保存ボタン
- 過去メモの履歴表示（日時、担当者名、内容）

## ページ1.2: 口座プロファイルページ

パス: `/accounts/:accountId`

### 概要
指定口座の全体像を把握するためのプロファイルページ。

### 口座情報ヘッダー
- 口座ID（大きなタイトル表示）
- 銀行名、エンティティ名、エンティティ種別（accounts.csvから取得）
- 口座の不正リスクスコア（その口座に関連する不正取引の割合から算出）: 色付きバッジで表示

### サマリーカードエリア（横4カラム）
- 総取引件数（送金+受取）
- 送金合計額
- 受取合計額
- 不正フラグ付き取引件数

### 取引金額推移グラフ
- 折れ線グラフ: X軸=日付、Y軸=取引金額
- 2系列: 送金額（青）、受取額（緑）
- 不正取引発生日にはマーカー（赤い点）を表示
- 期間フィルター: 日付範囲ピッカー

### 取引先一覧テーブル
- タブ切り替え:「送金先」/「送金元」
- カラム: 相手口座ID、相手銀行名、取引回数、合計金額、直近取引日、不正取引有無
- 不正取引を含む行は赤アイコンで表示
- 口座IDクリックで相手の口座プロファイルページへ遷移
- ソート: 各カラムヘッダークリックで昇順/降順

### 関連取引テーブル
- この口座が関わる全取引の一覧
- カラム: 日時、方向（送金/受取）、相手銀行、相手口座、金額、通貨、支払手段、不正フラグ
- フィルター: 方向、通貨、支払手段、不正フラグ
- ページネーション: 1ページ20件

## ページ2: アラート一覧ページ

パス: `/alerts`

### 概要
モデルが不正と判定した取引をリスト表示し、対応状況を管理するページ。

### ヘッダーエリア
- ページタイトル「アラート一覧」
- サマリーカード（横4カラム）:
  - 全アラート件数
  - 未対応件数（赤背景）
  - 調査中件数（黄背景）
  - 本日の新規アラート件数

### フィルターバー
- ステータスフィルター: チップ形式で「全て」「未対応」「調査中」「対応済み」「誤検知」を選択（複数選択可）
- 送金元銀行: ドロップダウン（検索付き）
- 送金先銀行: ドロップダウン（検索付き）
- 金額範囲: 最小値・最大値の入力フィールド
- 通貨: ドロップダウン
- 支払手段: ドロップダウン
- 期間: 日付範囲ピッカー
- 不正スコア範囲: レンジスライダー（0%〜100%）
- フィルターリセットボタン

### アラートテーブル
- カラム:
  - ステータス（色付きドットアイコン: 赤=未対応、黄=調査中、緑=対応済み、グレー=誤検知）
  - 不正スコア（パーセント表示 + ミニプログレスバー）
  - 取引日時
  - 送金元銀行 / 口座
  - 送金先銀行 / 口座
  - 金額
  - 通貨
  - 支払手段
- デフォルトソート: 不正スコア降順
- 各カラムヘッダークリックでソート切り替え
- 行クリック: 取引詳細ページ（1.1）へ遷移
- ステータス変更: ステータスドットクリックでドロップダウンメニュー表示、その場でステータス変更可能

### ページネーション
- 1ページ20件表示
- ページ番号ナビゲーション + 「前へ」「次へ」ボタン
- 1ページあたりの表示件数切り替え（20 / 50 / 100）

## ページ3: パターン分析ページ

パス: `/analytics`

### 概要
不正取引の傾向やパターンを統計的に可視化し、分析を支援するページ。

### 期間フィルター（ページ上部）
- 日付範囲ピッカー（開始日・終了日）
- クイック選択ボタン: 「直近7日」「直近14日」「全期間」

### セクション1: 時間帯分析
- ヒートマップ: X軸=時間帯（0〜23時）、Y軸=曜日（月〜日）、セルの色=不正取引件数
- カラーグラデーション: 薄い黄色（少）→ 赤（多）
- セルホバー時: 具体的な件数をツールチップ表示

### セクション2: 支払手段×通貨クロス集計
- テーブル形式: 行=支払手段（7種）、列=通貨（15種）、セル=不正率（%）
- 不正率の高いセルをヒートマップ色で着色
- セルクリック: 該当条件のアラート一覧へフィルター付き遷移

### セクション3: 高リスク銀行ランキング
- 横棒グラフ: 上位10銀行の不正取引件数
- 棒グラフの横に銀行名（accounts.csvから取得）と件数を表示
- タブ切り替え:「件数ベース」/「金額ベース」
- 棒クリック: 該当銀行のアラート一覧へフィルター付き遷移

### セクション4: モデル特徴量重要度
- 横棒グラフ: 全特徴量の重要度スコアを降順表示
- 各棒にパーセント値を表示

### セクション5: 不正パターン種別分布
- ドーナツチャート: Patterns.txtの8種パターン（FAN-OUT, FAN-IN, CYCLE, RANDOM, BIPARTITE, STACK, SCATTER-GATHER, GATHER-SCATTER）の件数分布
- チャート中央に合計パターン数を表示
- 凡例: パターン名と件数を右側に表示
- セグメントクリック: 該当パターンの取引一覧を下部に展開表示
